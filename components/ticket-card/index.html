TicketCard

공연/이벤트 티켓 정보를 시각적으로 제공하는 카드 컴포넌트로, SDUI 기반 실시간 상태 연동 및 다양한 뷰 타입에 대응합니다.

마지막 업데이트: 2025. 6. 1.

TicketCard

🎯 개요 및 목적

TicketCard는 공연, 전시, 이벤트 등 다양한 콘텐츠의 정보를 시각적으로 전달하기 위한 카드형 컴포넌트입니다. 기본 일정, 장소, 가격, 상태 정보 외에도 사용자의 행동을 유도할 수 있는 메타데이터를 포함하며, SDUI 형태로 서버에서 렌더링 구성을 제어할 수 있도록 설계되었습니다.

🧱 구조 및 설계 개요

BaseCard 기반으로 설계되어, preview / header / metadata / action 등의 Slot이 계층적으로 배치됩니다.

  • 일정 정보: 날짜, 기간, 요일 등 (startDate / endDate)
  • 장소 정보: venue
  • 상태 정보: 판매중, 매진, 종료 등
  • 가격 정보: 가격 범위 또는 최저가

👥 역할별 사용 가이드

🎨 디자이너

  • 상태별 컬러 및 아이콘 정의 (판매중, 매진, 종료 등)
  • 티켓 정보 시각 레이아웃 설계
  • Compact 타입 대응 (1줄 요약형)

📋 PM

  • 상태 정책 설정 (예매 가능 여부, 잔여 수량 등)
  • 할인/이벤트 정보 연동 기획
  • 실시간 상태 변화 정책 정의

💻 개발자

  • 실시간 상태 렌더링 연동 (saleStatus 기준)
  • 날짜 포맷팅 처리 (2025-06-03T19:00:00 → 6월 3일 (화) 19:00 등)
  • 메타데이터 처리 및 ViewFragment 구성

⚙️ 주요 Props

  • data: 티켓 정보 전체 데이터
  • variant: default 또는 compact
  • showStatus: 상태 뱃지 노출 여부
  • showMetadata: 메타데이터 표시 여부
  • onClick: 카드 클릭 시 호출할 핸들러
  • onBookmark: 북마크 토글 핸들러 (선택)

⚠️ 예외 상황 처리

  • saleStatus: soldout일 경우 카드에 "매진" 표시 및 인터랙션 비활성화
  • 이미지 로딩 실패 시 placeholder 노출 및 alt 텍스트 처리
  • 필수 필드 누락 시 카드 전체 숨김 또는 비노출 처리

⚡ 성능 고려사항

  • 이미지 lazy loading 필수
  • 리스트형 화면에서 Virtual Scroll 적용 권장
  • 예약 수량 표시 시 실시간 렌더링은 polling or socket 기준

♿ 접근성 가이드

  • aria-label에 이벤트명, 장소, 상태 포함
  • role="button", aria-pressed 등 상호작용 요소에 명시
  • 키보드 Tab → 카드 → 북마크 버튼 순서 포커스 이동

💻 실제 구현 예시

interface TicketCardProps {
  data: TicketCardData;
  variant?: 'default' | 'compact';
  showStatus?: boolean;
  showMetadata?: boolean;
  onClick?: (eventId: string) => void;
  onBookmark?: (eventId: string) => void;
}

const TicketCard: React.FC<TicketCardProps> = ({ data, variant = 'default', onClick, onBookmark }) => {
  return (
    <BaseCard onClick={() => onClick?.(data.eventId)}>
      {/* 티켓 이미지, 정보, 상태, 날짜 등 Slot 구성 */}
    </BaseCard>
  );
};

🧪 테스트 시나리오

  • 상태별 뱃지 노출 (available, soldout, ended 등)
  • 날짜 포맷 출력 확인
  • 북마크 클릭 시 이벤트 핸들링 여부 확인
  • Lazy Load 이미지 처리 및 fallback 확인
  • Compact 타입 시 정보 요약 노출 여부 확인

🔄 마이그레이션 가이드

기존 티켓 컴포넌트에서 다음 필드 추가 및 구조 개선 필요:

  • 날짜 필드는 startDate / endDate 기준 정규화
  • 가격은 min/max 범위와 통화 기준으로 통합
  • 상태는 saleStatus 중심으로 통일
  • 썸네일은 full URL 제공

✅ 개발자 체크리스트

  • 날짜 포맷 정규화 함수 제공 필요
  • saleStatus가 soldout일 경우 전체 disable 처리
  • onBookmark 토글 상태 UI와 동기화
  • 이벤트 ID 기준으로 라우팅 구성 또는 클릭 처리

🔚 향후 고려 사항

  • 예약 마감 임박 시 강조 스타일 적용
  • 북마크 상태 서버 동기화 또는 로컬 캐시 처리
