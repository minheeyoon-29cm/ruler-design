# Color 

컬러 시스템은 브랜드와 서비스의 일관성을 유지하면서도 
다양한 디자인 환경에 유연하게 대응할 수 있도록 설계되었습니다. 이를 통해 개인의 주관적인 판단에 의한 색상 선택이 아닌, 체계적으로 관리된 색상을 일관되게 사용할 수 있습니다.

29CM의 **Ruler 디자인 시스템은 일관성, 재사용성, 협업 효율성, 
그리고 확장성을 목표**로, **디자인 토큰**을 활용하여 컬러 시스템을 관리하고 있습니다. 
이를 통해 디자이너와 개발자가 동일한 기준을 바탕으로 효율적으로 협업하며, 
여러 플랫폼과 상황에서도 일관된 색상 경험을 제공할 수 있습니다.

---

### Scale Color Tokens

Scale Color Tokens는 색상 팔레트에서 사용되는 기본적인 색상 값을 정의하는 데 사용됩니다. 
이러한 토큰은 일반적으로 색상의 명도 단계(예: 밝음에서 어둠으로)를 제공하여 디자인 시스템 내에서 일관된 색상 체계를 유지하는 데 도움을 줍니다.

각 색상은 UI의 다양한 구성 요소에 적용할 수 있으며, 주로 배경, 텍스트, 경계 등의 시각적 요소에 사용됩니다. 
이런 색상은 특정 컨텍스트에 맞추기보다는 전체적인 디자인에 유연하게 적용될 수 있는 기본 색상입니다.

### Semantic Color Tokens

Semantic Color Tokens는 UI에서 특정 의미를 전달하기 위해 사용되는 색상 값을 정의합니다. 
이들은 사용자 경험의 맥락에서 더 구체적인 의미를 가지며, 상호작용, 경고, 성공 등과 같은 특정 상태를 표현하는 데 사용됩니다.

 특정 기능적 의미를 가진 UI 요소들, 예를 들어 버튼, 경고 메시지, 알림 등에서 사용됩니다. 예를 들어, 'primary' 색상 토큰은 주요 작업 버튼을 강조하는 데 사용되며, 
 'negative' 색상 토큰은 오류 상태를 나타내는 데 사용될 수 있습니다. Semantic 토큰은 기능적 의미를 강조하여 사용자에게 명확한 시각적 피드백을 제공합니다.## Props
