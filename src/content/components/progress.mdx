# Ruler Progress

### Foundation

디자인 시스템에서 Foundation 시스템은 시스템의 핵심을 형성하며,
디자인 토큰은 이 시스템의 중요한 구성 요소입니다. Foundation 시스템은 디자인 시스템의 기초를 제공하고,
디자인 토큰은 이 기초를 코드로 구현하여 일관성과 재사용성을 확보합니다.

| type | Design | FE | iOS | Android |
|---|---|---|---|---|

### Core Component

Core Component는 거의 모든 화면에서 반복적으로 사용되는 컴포넌트로, 단일 목적을 가진 간단하고 목적이 명확한 컴포넌트입니다.
이는 서비스 전반에 걸쳐 일관성을 유지하는 데 중요한 역할을 합니다. 거의 모든 페이지나 모듈에서 사용될 가능성이 높습니다.

 item.design.includes(STATUS.DONE)).length,
      coreComponents.filter((item) => item.fe.includes(STATUS.EMOTION_DONE)).length,
      coreComponents.filter((item) => item.fe.includes(STATUS.TAILWIND_DONE)).length,
      coreComponents.filter((item) => item.ios.includes(STATUS.DONE)).length,
      coreComponents.filter((item) => item.android.includes(STATUS.DONE)).length,
    ],
  ]}
  sumNumber={coreComponents.length}
/>

### Pattern Component

Pattern Component는 특정 시나리오나 화면에서 주로 사용됩니다. 여러 Core Component**를 조합하거나 특정 기능을 구현하기 위해 만들어진 복합적인 컴포넌트이며,
특정 요구 사항을 충족시키기 위해 설계되기 때문에 사용되는 경우가 좀 더 제한적입니다.**

 item.design.includes(STATUS.DONE)).length,
      patternComponents.filter((item) => item.fe.includes(STATUS.EMOTION_DONE)).length,
      patternComponents.filter((item) => item.fe.includes(STATUS.TAILWIND_DONE)).length,
      patternComponents.filter((item) => item.ios.includes(STATUS.DONE)).length,
      patternComponents.filter((item) => item.android.includes(STATUS.DONE)).length,
    ],
  ]}
  sumNumber={patternComponents.length}
/>
