# Toggle Icon Button

두 가지 상태(종종 켜짐 또는 꺼짐) 사이를 전환하는 데 사용되는 컨트롤입니다.

## Contents Guide


toggled icon button은 toggle이 가진 On - Off의 기능적인 맥락을 Icon 만으로 충분히 설명할 수 있어야 합니다.


![toggled 했을 때 아이콘이 변경되는 경우도 허용한다는 맥락](/images/components/toggle-icon-button/us-togglebtn-01.jpg)


대부분의 경우 하나의 아이콘 형태에서 Outlined 와 Fill 스타일 변경만으로 다양한 On - Off 의 맥락을 전달할 수 있지만,  toggle 되는 기능의 맥락을 더 명확히 사용자에게 전달하기 위해 아이콘 형태가 함께 변경되는 경우도 있습니다. 


프로덕트 디자이너의 의도에 따라 자유롭게 사용할 수 있도록 두 가지 방식 모두 허용합니다. 


다만 이는 아이콘 형태 자체가 toggle과 유사한 성질이나 개념을 가지고 있을 때 표현 가능한 방식입니다. (ex.자물쇠가 풀리거나 잠긴다 | 눈을 감아 안 보이고 - 떠서 보인다) 모든 아이콘이 이런 성질이나 개념을 가지기 어렵기 때문에 기본적으로 하나의 형태가 Outlined - Fill 로 toggle을 표현하게 되어 있습니다. 


### ✅ Do

![표현되는 아이콘 각각의 메타포간의 맥락이 있게 사용할 것](/images/components/toggle-icon-button/us-togglebtn-02-do.jpg)

### ❌ Don't

![표현되는 아이콘 각각의 메타포간의 맥락이 없을 시 사용에 주의할 것](/images/components/toggle-icon-button/us-togglebtn-02-dont.jpg)


toggle되는 기능에 표현되는 아이콘 각각의 메타포가 충분히 서로 연결성이 있고, 사용자가 그 맥락을 직관적으로 인지 가능한지 고려해주세요.


### ✅ Do

![Toggled Icon Button on-off 시 아이콘의 형태가 바뀌는 경우 맥락을 포함할 것](/images/components/toggle-icon-button/us-togglebtn-03-do.jpg)

### ❌ Don't

![Toggled Icon Button on-off 시 아이콘의 형태가 바뀌는 경우 맥락없이 쓰지말 것](/images/components/toggle-icon-button/us-togglebtn-03-dont.jpg)


Toggled Icon Button시 아이콘의 형태가 바뀌는 경우 사용자는 Toggle 이전의 맥락을 알기 어려울 수 있습니다. Label이 포함된 Toggled Button을 사용해주시거나 변경되는 맥락을 함께 명시하여 주는 것이 더 좋습니다.


## 미리보기


## Appearance


![Appearance](/images/components/toggle-icon-button/preview-toggle-ic-btn-00.jpg)


toggle icon button은 총 2가지의 Appearance(외형) 스타일을 가지며 29CM의 다양한 페이지와 콘텐츠에 따라 적재적소하게 사용할 수 있습니다.


a. default : 배경색을 가지고 있고 아이콘 사용시 기본 설정된 스타일 입니다.
b. isTransparent : 투명한 배경을 가지고 있는 스타일입니다


![Appearance](/images/components/toggle-icon-button/preview-toggle-ic-btn-00-01.jpg)


또한 toggle icon button에는 White 색상의 스타일이 제공됩니다. 이 스타일은 Theme(라이트,다크) 변화에도 White를 유지합니다.


![Appearance](/images/components/toggle-icon-button/preview-toggle-ic-btn-00-02.gif)


White 색상은 주로 이미지나 다른 색상의 배경이 있는 표면위에서 사용할 수 있습니다.


## Selected


![icon button Appearance](/images/components/toggle-icon-button/preview-toggle-ic-btn-01.jpg)


아이콘 내부가 채워진(fill) 스타일로 Selected 상태를 나타냅니다. Default 스타일에 경우 배경과 아이콘 색상이 함께 변경됩니다.


## Disabled


![toggle icon button의 비활성화-Disabled 상태](/images/components/toggle-icon-button/preview-toggle-ic-btn-02.jpg)


비활성화 상태는 현재 어떤 이유로 편집, 변경이 불가하여 사용할 수 없는 상태를 나타냅니다. 이후 해당 항목을 다시 사용할 수 있다는 맥락을 사용자에게 전달할 수 있습니다.



## Size


![Toggle Icon Button의 사이즈](/images/components/toggle-icon-button/preview-toggle-ic-btn-03.jpg)


Toggle Icon Button은 4가지 크기를 제공합니다. 크기는 표현해야할 정보 구성에 따른 계층에 유의하여 사용해야 합니다


## Behavior


![Toggle Icon Button WEB Behavior](/images/components/toggle-icon-button/preview-toggle-ic-btn-04.jpg)


### WEB


기본적으로 버튼에는 hover와 pressed와 같이 사용자 행동으로 결정되는 속성들이 포함되어 있습니다. 이러한 속성은 사용자에게 즉각적인 피드백을 제공하며, 클릭 가능한 요소를 명확하게 기술하는 역할을 합니다


![Toggle Icon Button Mobile Behavior](/images/components/toggle-icon-button/preview-toggle-ic-btn-05.jpg)


### Mobile


모바일에 경우 hovered 속성은 생략하고 pressed 속성으로 피드백을 제공합니다


![Keyboard Focus](/images/components/toggle-icon-button/preview-toggle-ic-btn-06.jpg)


### Keyboard Focus


Toggle Icon Button을 키보드 Tab키로 탐색하고 Space 또는 Enter키를 통하여 Selected 상태를 전환할 수 있습니다. 시각적으로는 파랑색 테두리를 추가하여 표현합니다.

## Programmatic State


사용자 맥락에 의해 개발자가 코드로 조작하거나 설정하는 상태를 의미합니다.


| State | Value | Default | Description |
| --- | --- | --- | --- |
| isDisabled | boolean | false | true일 때 비활성화 되어 선택할 수 없음 |
| isSelected | boolean | false | 사용자 선택 시 true로 전환 |


## Behavior State


주로 사용자 인터랙션과 관련이 있으며, 디자이너가 사용자 경험을 디자인할 때 중요한 역할을 합니다. 사용자는 이 상태를 직접적으로 경험할 수 있으며, 이를 통해 직관적이고 즉각적인 피드백 경험을 줄 수 있습니다.


| State | Value | Default | Description |
| --- | --- | --- | --- |
| hover | boolean | false | 마우스 오버 시 true로 전환 |
| pressed | boolean | false | 클릭 또는 탭시에 true로 전환 |
| focus | boolean | false | 키보드 포커싱 시 true로 전환 |
