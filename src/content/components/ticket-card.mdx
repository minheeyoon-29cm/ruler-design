TicketCard는 공연, 전시, 이벤트 등 다양한 콘텐츠의 정보를 시각적으로 전달하기 위한 카드형 컴포넌트입니다. 
기본 일정, 장소, 가격, 상태 정보 외에도 사용자의 행동을 유도할 수 있는 메타데이터를 포함하며, SDUI 형태로 서버에서 렌더링 구성을 제어할 수 있도록 설계되었습니다.

### UI ↔ API 매핑

![티켓카드 기본](https://img.29cm.co.kr/next-edition/2025/05/29/23ea9f55b41c467ba068c4c3642bb50c_20250529183818.jpg)



1. **랭킹 넘버**

   * API: 없음 (`rank` 없음)
   * 정책: 필요 시 클라이언트 측에서 정렬 후 부여 (ex. 인기순 등)
   * 분류: **비즈니스 정책**

2. **공연 이미지 (1:1 비율)**

   * API: `thumbnailUrl`
   * 정책: 1:1 비율 유지, full URL 사용
   * 분류: **디자인 정책**

3. **상단 뱃지 (ex: 이굿위크, 추천 등)**

   * API: `imageBadges[]`
   * 정책: 최대 3개, `variant`, `size`로 스타일 지정
   * 분류: **디자인 + 개발 정책**

4. **좋아요 상태 (찜)**

   * API: `isLiked`, `likeCount`
   * 정책: 상태형 아이콘 + 수치 함께 노출
   * 분류: **기능 정책**

5. **공연 제목**

   * API: `title`
   * 정책: 2줄 말줄임 처리 필요
   * 분류: **디자인 정책**

6. **공연장소 정보 (지역 · 공연장)**

   * API: `location`, `venue`
   * 정책: bullet(·)로 구분, 공간 줄바꿈 없이 1줄 처리
   * 분류: **디자인 정책**

7. **공연 일정**

   * API: `period`
   * 정책: 기간 또는 단일 날짜 (`YY.MM.DD ~ YY.MM.DD` or `YY.MM.DD`)
   * 분류: **비즈니스 정책**

8. **텍스트 하단 뱃지 (ex: 단독판매, 선예매)**

   * API: `textBadges[]`
   * 정책: 최대 4개, `variant`, `size`로 스타일 분기
   * 분류: **디자인 + 개발 정책**

9. **하단 정보 (찜 수, 리뷰 평점, 리뷰 수)**

   * API: `likeCount`, `reviewScore`, `reviewCount`
   * 정책:

     * `reviewScore`: 소수 1자리 (예: 4.2)
     * `reviewCount`: 괄호 표기 및 천단위 콤마
   * 분류: **디자인 정책**
