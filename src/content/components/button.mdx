# Button

사용자의 클릭 또는 터치를 통해 요구한 작업을 즉시 시작시키는 컨트롤입니다.
버튼을 통해 사용자는 취소 및 선택, 페이지 이동과 같은 작업 수행에 조치를 취할 수 있습니다.

## 사용법

수정필요: component_usage_demo 컴포넌트는 수동 변환 필요 

Button은 29CM 서비스 내에서 일반적으로 사용되며 가장 중요한 작업 수행을 돕는 버튼입니다. 화면의 흐름에서 사용자에게 예측가능하고, 필요한 곳에 적절히 버튼을 위치시키세요. 버튼을 통해 사용자가 화면에서 특정 조치를 취하거나 선택하며 흐름을 이어나갈 수 있도록 도와주어야 합니다.

## Contents Guide

버튼은 사용자가 버튼을 선택할 때 어떤 동작이 일어날지 예측할 수 있어야 합니다.
행동을 장려하는 동사로 사용자를 이끌어주세요.

### ✅ Do

![하나의 의도를 일관성 있게 전달할 수 있는 이미지](/images/components/button/usage/contents-guide-1-do.jpg)

### ❌ Don't

![하나의 의도를 일관성 있게 전달할 수 있는 이미지](/images/components/button/usage/contents-guide-1-dont.jpg)

사용자에 따라 해석의 여지가 다양한 표현보다는 하나의 의도를 일관성 있게 전달할 수 있는 문구로 작성해주세요.

서비스에서 일어나는 공통적인 사용자의 행동에는 하나의 언어를 사용해주세요.
또한 버튼 레이블에 NOW, 바로, 지금과 같은 문구는 고심해서 작성해주세요
경우에 따라서 즉시성을 띄우는 문구는 사용자에게 심리적인 부담과 버튼을 누르기 전 허들로 다가갈 수 있습니다.

버튼 레이블에 영문을 사용해야하는 경우, 대소문자를 함께 작성해주세요.
영미권 국가에서 대소문자 구분은 가독성을 위한 구분이자 언어적인 약속입니다.

## 미리보기

![button 종류를 한 눈에 보여주는 커버](/images/components/button/preview/button-preview-00-cover.jpg)

Button은 사용자가 스스로 어떤 작업을 수행할지 빠르게 파악할 수 있도록 위계와 버튼 내부 텍스트의 정보 이해를 돕는 아이콘을 지원하고 있습니다. 

## Hierarchy

![button prioirty](/images/components/button/preview/button-preview-01-hierarchy.jpg)

버튼의 Hierarchy는 서비스의 흐름 맥락에서 어떤 액션이 더 중요한지 강조하여 사용자의 작업 수행을 돕습니다.

a. primary : 사용자가 작업을 완료하는데 가장 중요한 액션에 사용됩니다. 강한 강조를 띄우고 있으므로 페이지 안에서 아껴서 사용해야 합니다. 
b. secondary : 약한 강조의 성격을 가지고 있어 primary보다 낮은 중요도의 사용자 액션에 사용될 수 있습니다. 
c. tertiary : 낮은 강조의 성격으로 주로 취소, 해제, 건너뛰기와 같이 사용자의 목표 달성과 무관한 액션에 사용될 수 있습니다. 

## Disabled

![button의 비활성화-Disabled 상태](/images/components/button/preview/button-preview-02-disabled.jpg)

비활성화 상태는 현재 어떤 이유로 편집, 변경이 불가하여 사용할 수 없는 상태를 나타냅니다. 이후 해당 항목을 다시 사용할 수 있다는 맥락을 사용자에게 전달할 수 있습니다.

## Pending

![button의 대기중-Pending 상태](/images/components/button/preview/button-preview-03-pending.jpg)

종종 버튼은 복잡한 서버의 설정과 통신의 처리 진행이 이루어지고 있음을 나타낼 수 있습니다. 이 경우 버튼 context에 spinner가 표현됩니다. pending상태는 사용자에게 좋은 경험이 아니므로 신중하게 사용해야 합니다.

## Negative

![button의 부정적이거나 파괴적인 성격을 나타내는 Negative 상태](/images/components/button/preview/button-preview-04-negative.jpg)

사용자가 반드시 알아야 하는 크리티컬한 액션인 경우 Negative를 사용하여 해당 내용을 강하게 인지시킬 수 있습니다. 주로 사용자에게 중요한 정보가 삭제되거나 변경되는 경우에 사용됩니다. Negative 는 사용자에게 좋은 경험이 아니므로 신중하게 사용해야 합니다. 

## Size

![button의 4가지 크기](/images/components/button/preview/button-preview-05-size.jpg)

Button은 4가지 크기를 제공하며 Medium이 기본 값으로 자주 사용됩니다, 크기는 표현해야할 정보 구성에 따른 계층에 유의하여 사용해야 합니다

### Min Width

레이블 길이가 짧을 경우를 대비하여 모든 버튼에는 Minimum Width가 포함되어 있습니다. 버튼은 이보다 작은 가로폭을 가질 수 없습니다.

## Icon

수정필요: multi_image 컴포넌트는 수동 변환 필요 

버튼과 함께 사용되는 아이콘은 각 위치마다 역할이 정해져 있습니다. prefix(접두사)와 postfix(접미사)의 의미를 주의하여 사용해주세요.

![Prefix Icon 사례 설명](/images/components/button/preview/button-preview-07-prefixicon.jpg)

### Prefix Icon

prefix icon은 레이블 왼쪽에 위치하며 레이블의 텍스트의 의미를 강조하거나 보조할 때 사용합니다.

![postfix Icon 사례 설명](/images/components/button/preview/button-preview-08-actionicon.jpg)

### Postfix Icon

postfix icon은 레이블 오른쪽에 위치하며 사용자의 행동과 상호 작용을 암시합니다.  페이지 이동, 바텀시트 등장이 있을 수 있습니다. 때문에 사용자가 기대하는 액션이 무엇인지 고려하여 사용하길 권장합니다.

## Programmatic State

사용자 맥락에 의해 개발자가 코드로 조작하거나 설정하는 상태를 의미합니다.

| State | Value | Default | Description |
| --- | --- | --- | --- |
| isDisabled | boolean | false | true일 때 비활성화 되어 선택할 수 없음 |
| isPending | boolean | false | true일 경우 Label대신 spinner가 나타남 |

## Behavior State

주로 사용자 인터랙션과 관련이 있으며, 디자이너가 사용자 경험을 디자인할 때 중요한 역할을 합니다. 사용자는 이 상태를 직접적으로 경험할 수 있으며, 이를 통해 직관적이고 즉각적인 피드백 경험을 줄 수 있습니다.

| State | Value | Default | Description |
| --- | --- | --- | --- |
| hover | boolean | false | 마우스 오버 시 true로 전환 |
| pressed | boolean | false | 클릭 또는 탭시에 true로 전환 |
| focus | boolean | false | 키보드 포커싱 시 true로 전환 |
