# Search Field

사용자가 키워드나 문구를 직접 입력하여 관련 정보를 얻기 위해 사용합니다.

## 사용법


수정필요: component_usage_demo 컴포넌트는 수동 변환 필요 


Search Field 를 통해 사용자는 원하는 항목을 직접 검색하고 정보를 필터링 할 수 있습니다.

### ✅ Do

![Search Field의 기본 상태에서는 텍스트 필드 내부에 문구가 포함되어야 함](/images/components/search-field/usage/search-field-do-01.jpg)

### ❌ Don't

모든 Search Field의 기본 상태에서는 텍스트 필드 내부에 문구가 포함되어야 합니다. 기본적으로 “Search” 가 표기 되어 사용자에게 검색이 가능함을 인지시켜줍니다.


### ✅ Do

![상황에 따라 적절하게 사용자의 검색을 돕는 구체적인 키워드를 포함할 수 있음](/images/components/search-field/usage/search-field-do-02.jpg)

### ❌ Don't

![지나친 마케팅 문구는 사용자가 우리를 속물처럼 느껴지게 함](/images/components/search-field/usage/search-field-dont-02.jpg)


상황에 따라 적절하게 사용자의 검색을 돕는 구체적인 키워드를 포함할 수 있습니다. 단 사용자의 탐색 목적의 제안이 아니라면 문구 사용을 고려해주세요. 영업 또는 판매에 대한 성격이 강한 문구는 오히려 사용자의 반감을 불러일으킬 수 있습니다.


### ✅ Do

![검색 결과에는 오류 상태가 없음](/images/components/search-field/usage/search-field-do-03.jpg)

### ❌ Don't

![검색 결과에는 오류 상태가 없음](/images/components/search-field/usage/search-field-dont-03.jpg)


검색 결과에는 오류 상태가 없습니다. 기본적으로 검색 기능에는 검색어의 철자 오류, 다양한 예외 케이스들을 예상하고 사용자의 모든 키워드들을 수용할 수 있어야 합니다. 


![검색 결과 없을 시 대안 : 사용자가 다시 탐색을 이어 나갈 수 있도록 구체적인 도움을 줄수록 좋음](/images/components/search-field/usage/search-field-do-04.jpg)


위와 같은 경우 '검색 결과가 없음'에 대한 페이지를 표시하고 사용자가 다시 탐색을 이어 나갈 수 있도록 구체적인 도움을 줄 수 있도록 하면 좋습니다. 대안을 마려한거나 , 다른 키워드를 제시하거나, 다른 상품을 추천하는 등 다양한 방법으로 사용자의 목적 달성, 탐색에 대한 의지를 계속 북돋아 주세요.


## 미리보기


![Search Field Part Name](/images/components/search-field/preview/search-field-preview-00-part.jpg)


Search Field 는 크게 3개의 영역으로 구성됩니다. 각 영역은 다른 컴포넌트와 달리 영역마다 들어가 있는 아이콘이 고정되어 있으며 다른 아이콘이나 액션을 허용하지 않고 있습니다.


a. prefixIcon : 필드 내부에 문구들이 검색과 관련있는 맥락임을 인지시키는 기호 언어로서 사용됩니다. 돋보기 아이콘 외에 다른 아이콘은 사용할 수 없습니다.
b. placeholder | search word : 사용자의 검색을 돕는 placeholder, 사용자의 검색어가 입력되어 나타납니다. 
c. postfixItem : 기본 상태일 때는 노출되지 않으며 사용자가 검색을 시작할 때, 검색을 완료했을 때 나타나 검색어를 지울 수 있는 액션이 나타납니다.

### WEB


Search Field에도 hover와 pressed, Focus 같이 사용자 행동으로 결정되는 속성들이 포함되어 있습니다. 이러한 속성은 사용자에게 즉각적인 피드백을 제공하며, 클릭 가능한 요소, 활성화되어 사용가능한 상태를 명확하게 기술하는 역할을 합니다


수정필요: multi_image 컴포넌트는 수동 변환 필요 


### Mobile


모바일에 경우 hovered 속성은 생략하고 pressed 속성으로 피드백을 제공합니다


수정필요: multi_image 컴포넌트는 수동 변환 필요 


### Keyboard Focus


기본 상태에서 사용자가 검색을 시작하기위해 Tab을 사용하여 search field를 활성화 하는 파랑색 테두리를 추가합니다. 이미 검색어가 있는 경우, 검색어가 전체 선택되며 사용자가  텍스트를 다시 타이핑하거나 삭제할 수 있도록 합니다.

## Behavior State


주로 사용자 인터랙션과 관련이 있으며, 디자이너가 사용자 경험을 디자인할 때 중요한 역할을 합니다. 사용자는 이 상태를 직접적으로 경험할 수 있으며, 이를 통해 직관적이고 즉각적인 피드백 경험을 줄 수 있습니다.


| State | Value | Default | Description |
| --- | --- | --- | --- |
| hover | boolean | false | 마우스 오버 시 true로 전환 |
| pressed | boolean | false | 클릭 또는 탭시에 true로 전환 |
| focus | boolean | false | 키보드 포커싱 시 true로 전환 |
