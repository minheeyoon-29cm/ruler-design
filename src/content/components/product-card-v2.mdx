ProductCard는 29CM 내에서 파트너 브랜드의 상품을 Display하며, 구매를 원하는 고객이 판단에 필요한 정보를 명확히 인식할 수 있도록 구성된 카드형 컴포넌트입니다.

**핵심 변경사항 (v1 → v2)**

- SDUI 대응 구조로 재설계되어 다양한 뷰에서 동적으로 조합 가능
- 실시간 프로모션 배지 및 상태 표현 기능 강화
- 데이터 구조 최적화로 백엔드 API 스키마와 일치


### UI ↔ API 매핑

![ProductCard 기본](https://img.29cm.co.kr/next-edition/2025/05/29/b24b52cfa0e24dde9e8dd2c7ce7ec7af_20250529180944.jpg)

1. **랭킹 넘버**

   * API: `rank`
   * 정책: `nullable`, 없으면 UI에서 미노출
   * 분류: 비즈니스 정책

2. **상품 이미지 (1:1 비율)**

   * API: `thumbnailUrl`
   * 정책: 1:1 비율 유지 필요, Full URL 제공
   * 분류: 디자인 정책

3. **이미지 뱃지 (예: 이구위크, 뷰티위크 등)**

   * API: `imageBadges[]`
   * 정책: 최대 2개 - 권장하지 않음 
   * 분류: 디자인 + 개발 정책

4. **좋아요**

   * API: `없음`
   * 정책: 필요에 따라 노출, 비노출 가능 
   * 분류: 디자인 + 개발 정책 

4.5 **좋아요 상태 (찜)**

   * API: `isLiked`, `likeCount`
   * 정책: 상태형 아이콘 + 수치 함께 노출
   * 분류: 기능 정책

5. **브랜드명**

   * API: `brandName`
   * 정책: 브랜드 상세 페이지로 링크될 가능성
   * 분류: 디자인 정책

6. **상품명**

   * API: `productName`
   * 정책: 2줄 말줄임 처리, 고정 높이 권장
   * 분류: 디자인 정책

7. **가격 정보 (할인율 + 노출가 + 정가)**

   * API: `saleRate`, `displayPrice`, `originalPrice`
   * 정책: 할인율 0일 경우 미표시 가능, 계산 방식 명시 필요
   * 분류: 비즈니스 정책

8. **텍스트 하단 뱃지 (예: 무료배송, 쿠폰)**

   * API: `textBadges[]`
   * 정책: 최대 3개, 서버에서 내려주는 배열 순으로 나열
   * 분류: 디자인 + 개발 정책

9. **하단 정보 (찜 수, 리뷰 평점, 리뷰 수)**

   * API: `likeCount`, `reviewScore`, `reviewCount`
   * 정책: `reviewScore`는 소수점 1자리, `reviewCount`는 콤마 표시
   * 분류: 디자인 정책
