# Badge

Badge는 메타데이터를 표시하여 정보를 전달하거나 사용자의 관심을 유도하는 표시자입니다. 포함한 정보의 중요도에 따라 색상, 크기를 조정하여 사용자에게 전달할 수 있습니다.

![heroImage](https://img.29cm.co.kr/next-edition/2024/08/28/6f1e4a520c4b46dfbd07c612a2e800ab_20240828140607.jpg)

- 🅐 Root : Badge를 감싸는 Container
- 🅑 Label : Data를 표시하는 텍스트
- 🅒 Prefix : Label의 Data를 보조하는 아이콘 기호가 들어오는 영역

---

![Image](https://img.29cm.co.kr/next-edition/2024/08/28/d7097fd552ae436b9d141f47f2219ec7_20240828144136.jpg)

Badge는 크게 2가지의 Appearance(외형) 종류 가지며 정보의 성격과 사용되는 UI에 따라 용례가 제한됩니다.

### Defulat 

![Image](https://img.29cm.co.kr/next-edition/2024/08/28/d9f0f6c1156c438aa3cf9511c770e4b5_20240828145103.jpg)

Defulat는 Badge를 사용하면 기본적으로 설정되어 있는 스타일이며 
주로 Product Card의 상품 정보 표기 또는 페이지와 섹션의 그룹화 상품 목록들에 사용됩니다. 
랭킹, 쿠폰 사용 유무, 무료배송 등 상품의 정보 중 사용자의 구매 결정에 도움이 되는 정보를 나타내는 데 유용합니다.

### Tone 

![Image](https://img.29cm.co.kr/next-edition/2024/08/28/b6b93f5ecd5a4363b1aecfb480a8ddca_20240828150005.jpg)

Tone은 다양한 UI 메시지의 성격을 구분하여 사용자에게 더 명확하고 직관적인 피드백을 제공하기 위해 정의된 Props입니다. 
이 속성은 주로 UI의 알림, 상태, 경고 등을 시각적으로 구분하여 사용자의 주의를 효과적으로 유도하는 역할을 합니다.

![Image](https://img.29cm.co.kr/next-edition/2024/08/28/4f0a0106c1a14e6bb6cf330ef1a9445d_20240828150521.jpg)

각 Tone은 특정 상황에 맞는 색상과 메시지 스타일로 설정됩니다. 
이를 통해 사용자가 메시지를 보다 쉽게 이해하고, 상황에 적절하게 반응할 수 있도록 합니다.

- Default(기본) : 일반적인 정보 제공이나 안내 메시지를 표시할 때 사용됩니다.
- Information(정보) : 중요한 정보나 사용자에게 주의를 기울이도록 알릴 때 사용됩니다.
- Positive (긍정적 상태) : 긍정적인 피드백이나 성공적인 작업 완료를 알릴 때 사용됩니다.
- Negative (부정적 상태) : 오류나 문제가 발생했을 때 사용자에게 이를 알릴 때 사용됩니다.

### Priority 

![Image](https://img.29cm.co.kr/next-edition/2024/08/28/596dece6c6404032a8d19fd0123b7cbf_20240828152013.jpg)

Priority는 디자인 시스템에서 특정 UI 요소가 사용자에게 얼마나 중요한지를 나타내는 개념입니다. 
이는 사용자가 화면을 처음 봤을 때 어느 요소가 더 눈에 띄어야 하고, 어떤 정보가 가장 먼저 전달되어야 하는지를 결정합니다. 

- Primary: 가장 높은 우선순위를 가지며, 사용자에게 가장 눈에 띄는 스타일이 적용됩니다.
- Secondary: Primary보다는 덜 강조되지만 여전히 주목할 만한 중요성을 가집니다.
- Tertiary: 가장 낮은 우선순위를 가지며, 강조도가 매우 낮아 배경처럼 느껴질 수 있습니다.

*Tone에는 `tertiary`가 지원되지 않고 있습니다.

![Image](https://img.29cm.co.kr/next-edition/2024/08/28/38b9a1d0fd044727af2406a18cd9efbf_20240828153859.jpg)

Badge에는 `small`| `medium` 두 가지 사이즈를 제공합니다. 

### Prefix Icon 

![Image](https://img.29cm.co.kr/next-edition/2024/08/28/131976784faf4b2fa470fce13af095b9_20240828154017.jpg)

Badge는 Label과 Prefix로 구성되며 Prefix는 필요에 따라 추가하거나 감출 수 있습니다.
- 🅐 label : 뱃지의 필수 정보 값으로 Badge는 Label이 없어서는 안됩니다.
- 🅑 prefixIcon : 필요에 따라 Badge에 있을수도, 없을수도 있으며 label의 정보 인지를 돕는 아이콘이 들어올 수 있습니다.

### isBlur & isWhite
 

![Image](https://img.29cm.co.kr/next-edition/2024/08/28/7b3dd6df48484d8cb1f3e5dee81d4e81_20240828154707.jpg)

`isBlur`| `isWhite` 속성은 각각 Appearance에 해당되며 이미지 위, 특정 배경위에 올라가는 경우에 사용하는 스타일 속성입니다. 
별도의 Priority 를 가지지 않으며 아주 예외적으로 사용됩니다.

![Image](https://img.29cm.co.kr/next-edition/2024/08/28/0060b0ddef974003b7a24d222bf191bf_20240828155200.png)

Badge에 표현되는 정보나 상태에는 너무 긴 문구나 동사가 들어가는 것 보다 명료하게 요점만 표현하는 것이 사용자의 결정에 더 도움이 됩니다. 
영업 또는 판매의 의도가 담긴 문구는 단기적으로 좋은 효과를 가져올 수 있으나 장기적으로 노출되는 상품들의 분별력을 저하해 사용자에게 아무런 감흥을 불러일으키지 못하는 정보로 인지될 수 있습니다.
