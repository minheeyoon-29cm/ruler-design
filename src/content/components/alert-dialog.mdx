# Alert Dialog

Alert Dialog는 사용자의 여정에서 중요한 정보를 표시합니다. 사용자가 Alert Dialog에서의 작업을 완료할 때까지 다른 상호작용을 차단합니다.

## 사용법

Alert Dialog는 사용자에게 간단하거나 확인해야하는 중요한 메세지를 전달하고 응답 또는 선택을 요구하기 위해 의도적으로 사용자 여정 중간에 표시됩니다.

Title에는 주어나 사용자가 요청한 작업 내용에 대한 목적을 포함하는게 좋습니다. 또한 Description에도 사용자가 선택할 응답에 대한 충분한 맥락 서술이 필요합니다. Alert Dialog에서 선택한 응답이 어떤 작업으로 이어질지에 대해서 사용자가 충분히 가늠할 수 있도록 구성해 주세요.


<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/usage/ad-02.jpg" alt="Alert Dialog의 트리거 버튼과 연속성있는 문구를 사용하길 권장하는 가이드" />


사용자는 Alert Dialog를 보게 되면 이전 작업에 관한 내용을 확인하기 어렵습니다. 제목 작성이 어렵다면, 이전 작업에서 사용자가 어떤 작업을 수행했는지 떠올려 보거나 탭 한 Button의 Label와 일치하거나 유사한 단어를 포함하면 좋습니다.

<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/usage/ad-03-dont.jpg" alt="Alert Dialog Title에 사용자 선택을 되묻는 텍스트 작성보다 사용자가 내린 결정에 결과값을 알려줄 수 있도록 해야한다는 가이드" />


“정말 해지하겠어요?” 또는 “취소하시겠어요?”와 같은 질문은 이전 작업에서 사용자가 이미 내린 결정에 대해서 2번 고민하게 만들거나 불필요한 불안감을 조성할 수 있습니다. 이런 경우, 사용자가 내린 결정에 관한 결과 또는 효과에 초점을 맞추어 정보를 전달할 수 있도록 메시지를 고려해 주세요.

<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/usage/ad-04-do.jpg" alt="Alert Dialog에 Title만 또는 Description만 있는 경우는 사용해선 안된다는 가이드" />

### ❌ Don't

<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/usage/ad-04-dont.jpg" alt="Alert Dialog에 Title만 또는 Description만 있는 경우는 사용해선 안된다는 가이드" />


Alert Dialog의 Title만 있거나, Description만 있는 경우는 맥락 설명이 충분하지 않을 수 있습니다.
사용자에게 요구해야 할 요청이 있다면 요청할 작업과 요청한 이유 또는 작업을 통한 혜택이나 가치를 포함하여 메세지를
구성하는 것이 좋습니다. 


### ✅ Do

<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/usage/ad-05-do.jpg" alt="Alert Dialog Title은 2줄, Description는 3줄 이상 넘어가지 않도록 안내하는 가이드" />

### ❌ Don't

<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/usage/ad-05-dont.jpg" alt="Alert Dialog Title은 2줄, Description는 3줄 이상 넘어가지 않도록 안내하는 가이드" />


Title은 2줄, Description는 3줄 이상 넘어가지 않는 것이 좋습니다. 사용자가 인지해야 할 정보가 너무 많은 경우 메시지 전달력이 크게 떨어져 오히려 사용자의 선택을 어렵게 만듭니다. 사용자에게 필요하거나 유용한 정보를 간결하게 전달할 수 있도록 고려해 주세요.


## 미리보기


## Title


<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/preview/alert-dialog-preview-01-title.jpg" alt="Alert Dialog Title 사용 가이드" />


Alert Dialog는 항상 Title을 가지고 있어야 합니다.
Title은 사용자가 하려는 행동의 주요 효과나 선택 이 후에 상황과 결과 등 요점을 간결히 요약하여 전달할 수 있어야 합니다.
이를 통해 Alert Dialog가 나타난 목적에 대해 사용자가 빠르게 인지할 수 있습니다.
타이틀 문구는 최대 2줄이 넘어가지 않도록 작성해주세요.


## Description


<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/preview/alert-dialog-preview-02-text.jpg" alt="Alert Dialog Description 사용 가이드" />


Alert Dialog에는 Description가 항상 포함됩니다. Description은 사용자가 선택을 내리기위한 정보와 맥락을 전달합니다.
 Description은 최대 3줄이 넘어가지 않게 작성해주세요.


## Primary Action


<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/preview/alert-dialog-preview-03-primary-action.jpg" alt="Preview - Primary Action" />


Alert Dialog 내 정보나 맥락이 사용자의 단순 인지가 필요한 내용이거나, 그에 따른 중요한 선택지가 하나밖에 없다면
Primary Action을 사용합니다. 중요한 정보를 확실히 인지시키고 사용자에게 불필요한 선택지를 제공하지 않습니다.
또한 가벼운 내용이라면 Toast 메세지로도 처리가 가능한지 고려가 필요합니다.


<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/preview/alert-dialog-preview-04-primary-action.jpg" alt="Preview - Primary Action" />


Primary Action은 사용자가 서비스를 사용하는 흐름에서 목적을 달성하기 위한 가장 중요하거나 필요한 Action에 사용되어야 합니다.
흐름에서 사용자에게 가장 바라는 것은 무엇인지 고려하여 사용해주세요.


## Secondary Action


<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/preview/alert-dialog-preview-05-secondary-action.jpg" alt="Preview - Secondary Action" />


Secondary Action은 대부분 dismiss(해제)에 해당하는 선택지를 제공합니다.
Alert Dialog는 사용자의 작업 수행과정의 여정을 막는 동작을 동반하기 때문에
언제든 원래의 여정으로 돌아갈 수 있도록 도울 수 있어야합니다.


<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/preview/alert-dialog-preview-06-secondary-action.jpg" alt="Preview - Secondary Action2" />


Alert Dialog 의 정보와 맥락에 내용에 따라 Secondary Action에는 dismiss가 아닌 다른 선택지에 해당하는
액션이 허용될 수 있습니다. 이런 경우 Primary Action으로 해결되지 않는 사항에 대한 대안책을 명확히 작성해주세요.


## Eerro


<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/preview/alert-dialog-preview-07-error.jpg" alt="Preview - Eerro" />


서비스 통신 장애, 일시적인 오류 또는 서비스의 최신 업데이트와 같이 확인이 필요한 중요한 케이스에
Alert Dialog를 활용할 수 있습니다. 이런 경우 사용자에게 반드시 원인과 대안을 제시하는 정보와 맥락을 표시하고
그에 따른 액션을 명확히 제공해야 합니다.


## Destructive


<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/preview/alert-dialog-preview-08-destructive.jpg" alt="Preview - Destructive" />


Destructive Action은 사용자에게 부정적인 영향을 끼치는 경험에 해당하는 액션을 명확히 전달할 필요가 있을 때 사용합니다.
사용자의 선택으로 인해 사라지는 혜택과 겪게될 피해에 대한 명확한 안내를 함께 포함하여 사용해야하며,
극히 드문 경우에만 사용되어야 합니다.

tab시 Primary Action으로 바로 이동되며 2번째 Tab시 Secondary Action으로 이동됩니다.

### Voice


Alert Dialog가 나타났음을 알리고 Alert Dialog의 내용과 맥락을 알려줄 수 있어야 합니다.
Title과 Text의 내용을 알려준 후 Primary Action과 Secondary Action을 선택할 수 있도록 안내해주세요.
사용자가 혼선없이 본인에게 필요한 선택을 할 수 있도록 Alert Dialog가 화면에 나타난 이후에는
뒷 배경에 다른 화면과의 상호작용을 막아주세요.

## 상태


## Anatomy


<img src="https://raw.githubusercontent.com/29CM-Developers/ruler-static-contents/main/pages/component/alert-dialog/resources/web/states/alert-dialog-anatomy.jpg" alt="Alert Dialog Anatomy 이미지" />


a. Overlay : Alert Dialog 뒤에 표시되는 Overlay 배경 입니다
b. Container : Alert Dialog의 내용을 담는 배경입니다.
c. Contents container : 기본적으로 Title과 Description의 내용을 저장합니다.
d. Title: Alert Dialog의 타이틀 입니다. 전달하려는 내용의 핵심정보가 압축되어 작성됩니다.
e. Description : Title의 내용을 보조하는 문장입니다.
f. Action container : 기본적으로 Button을 담습니다.
g. Secondary action : 보조적인 액션을 표현합니다. 
h. Primary action : Alert Dialog 맥락에 따른 가장 중요한 액션을 표현합니다. 


## Programmatic State


사용자 맥락에 의해 개발자가 코드로 조작하거나 설정하는 상태를 의미합니다.


| State | Value | Default | Description |
| --- | --- | --- | --- |
| isOpen | boolean | false | Alert Dialog가 열려 있는지 여부 |
| open | () =&gt; void | false | Alert Dialog가 열림 |
| close | () =&gt; void | true | Alert Dialog가 닫힘 |
